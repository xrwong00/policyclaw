# UMHackathon 2026 — Reference Documents

Source documents from the UMHackathon 2026 organizers. Kept in the repo so Claude Code and teammates can reference them directly without chasing email attachments.

| File | What it is |
|---|---|
| `judging-criteria.pdf` | Preliminary-round scoring rubric — 14 weighted aspects across 4 categories (100% total). |
| `prd-sample.pdf` | Product Requirement Document template (sample uses "Stitch" as worked example). |
| `sad-sample.pdf` | System Analysis Documentation template — architecture, GLM-as-service-layer, NFRs, priority matrix. |
| `qatd-sample.pdf` | Quality Assurance Testing Documentation template — risk matrix, CI/CD gates, test cases, AI boundary tests. |

The distilled rubric (weights + judgment rules) lives in the repo root `CLAUDE.md` under the **"Hackathon judging"** section so it loads into every Claude Code session. Open the PDFs here only when you need the full text (e.g., drafting a deliverable).

**Mandatory-model rule (rubric p.3):** entries must use Z.AI GLM. PolicyClaw's `api.ilmu.ai` / `ilmu-glm-5.1` endpoint is an authorized Z.AI endpoint and satisfies this rule.
